# Моя конфигурация 1С

Учебный проект на платформе 1С:Предприятие 8.3.

## Реализовано:
- Справочники: Организации, Товары, Менеджеры, Филиалы, Города
- Документы: Поступление товаров, Продажа товаров
- Перечисления: Категории
- Управляемые формы для всех объектов

---

## Программирование и регистры

### 1. Модули объектов и программирование форм
В рамках модулей документов реализована бизнес-логика расчетов:

- **Программирование расчетов при учете поступлений товаров**  
  В модуле документа «Поступление товаров» реализованы алгоритмы подсчета итоговых сумм, налогов и стоимости партии.

- **Программирование расчетов при учете продажи товаров**  
  В модуле документа «Продажа товаров» настроена логика расчета себестоимости и валовой прибыли на основе текущих данных учета.

### 2. Регистры сведений
Реализован механизм хранения закупочных цен в разрезе времени:

- **Создание регистра сведений «Цены поступления»**  
  Регистр предназначен для определения закупочной цены товара. Данные хранятся развернутыми во времени (периодический регистр), что позволяет отслеживать изменение цен поставщиков.

- **Редактирование документа «Поступление товаров»**  
  В форму документа добавлена программная логика записи цен в регистр сведений при проведении документа.

- **Использование общего модуля**  
  Создан общий модуль для получения актуальных данных (цены на дату) из регистра сведений «Цены поступления», чтобы избежать дублирования кода.

- **Программирование формы документа «Продажи товаров»**  
  Форма документа доработана: при подборе товара автоматически подставляется закупочная цена из регистра сведений для расчета себестоимости и рентабельности.

### 3. Регистры накопления. Движения по регистрам
Организован количественный и аналитический учет движений товаров:

- **Создание регистра накопления «КоличествоТоваров»**  
  Регистр остатков для оперативного учета количества товаров на складах.

- **Редактирование документа «ПоступлениеТоваров»**  
  Настроено формирование движений по регистру «КоличествоТоваров» (приход) при проведении документа.

- **Редактирование документа «ПродажаТоваров»**  
  Настроено формирование движений по регистру «КоличествоТоваров» (расход) при проведении документа.

- **Создание оборотного регистра накопления «ПродажиПоФилиалам»**  
  Реализован регистр оборотов для учета реализации товаров в разрезе филиалов, что позволяет анализировать эффективность продаж по каждому подразделению.
